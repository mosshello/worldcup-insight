# 模型训练方案

## 当前结论

线上主方向仍由市场赔率去水、外部市场对比、固定权重上下文、走势规则和体彩多玩法分布组成。
项目另有 `elo-poisson-1.0` 统计影子模型：以2022世界杯和世界杯前近两年国家队比赛训练，
2026世界杯仅作按时间顺序的封存验证与赛内状态更新。影子模型不会覆盖线上市场主方向。

## 完整命令

```powershell
python predict.py train-worldcup --refresh
python predict.py model-report --json
python predict.py model-predict --home Argentina --away Egypt --neutral --json
```

原始数据和模型产物分别写入 `data/training/international_results.csv` 与
`data/training/statistical_model.json`。两者均记录来源、下载时间、哈希、时间边界和模型版本。

## 数据门槛

- 统计赛果模型最低实验门槛：基础训练不少于500场、连续时间验证不少于100场。
- 市场融合校准最低门槛：500场具有赛前赔率快照和90分钟赛果的比赛。
- 建议上线门槛：2,000 场以上，覆盖多个赛事和至少两个完整赛季/大赛周期。
- 每场必须保存：`data_as_of`、开赛时间、HAD/HHAD/TTG/HAFU、外部市场、阵容可用性、
  阶段、场地、休息天数、90分钟比分和收盘价。
- 禁止使用开赛后的赔率、赛中事件或最终赛果生成赛前特征。

## 第一阶段：可解释基线

1. Elo 估计赛前球队强度，并按比赛日期顺序更新；友谊赛、资格赛和大赛使用不同权重。
2. 双 Poisson 估计主客队进球均值；当前特征包括 Elo 差、滚动进攻/防守、赛事均值、
   主场和中立场。休息、伤停与阵容仍由实时融合层处理。
3. 用比分矩阵统一导出胜平负、大小球、双方进球和比分概率。
4. 市场去水概率作为独立基线，不直接当训练标签。

## 第二阶段：市场融合与校准

- 使用逻辑回归或受约束加权，将统计模型概率与市场概率融合。
- 只在验证集上选择权重；禁止按单场赛果事后调整。
- 使用 isotonic 或 Platt scaling 校准胜平负概率。
- 总进球模型单独校准，不能由最低比分SP替代。

## 时间切分

- 训练集：较早比赛。
- 验证集：训练集之后的连续时间段。
- 测试集：最新且完全封存的赛事。
- 禁止随机拆分，因为同一赛事、同一球队相邻比赛会造成信息泄漏。

## 验收指标

- 胜平负：Log Loss、Brier Score、分箱校准误差。
- 进球：Poisson deviance、总进球分布误差、大小球 Brier Score。
- 投注评估：收盘线价值、去水后收益、最大回撤；至少报告95%置信区间。
- 新模型必须同时优于“市场概率基线”和“当前透明启发式”，否则不替换生产版本。

## 上线流程

1. `python predict.py train-worldcup --refresh` 固化数据与模型版本。
2. `python predict.py audit-training --json` 必须返回零非法记录。
3. 对2026世界杯封存集只运行一次按时间顺序评估，禁止用测试指标反复调参。
4. 先影子运行，不影响推荐；补齐同时间戳市场快照并证明优于市场基线后再决定是否接管。
