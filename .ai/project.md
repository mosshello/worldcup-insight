# 世界杯控制台 MVP

## 项目定位
基于赛前可得数据，对世界杯比赛输出可解释的 90 分钟胜平负市场概率、上下文信号与回测结果。

## 技术约束
- 技术栈：Python 3.12，仅使用标准库。
- 架构边界：统一数据管理器调用供应商公开HTTPS API并输出规范化快照；分析器本身不得联网；本地JSON仅用于快照重放和回测；赛果字段只允许进入回测评价，不得进入预测计算；Polymarket与中国体彩网价格必须分栏保存，不得互相冒充或静默替代。
- 实时数据安全：运行时只允许无需密钥的匿名公开HTTPS读取接口；接口失败不得静默回退旧数据。
- 数据防泄漏：每份分析数据必须记录 `data_as_of`，且回测输入的截止时间必须早于开赛时间。
- 概率口径：保留赔率去水后的市场概率；上下文调整概率必须明确标注为未校准的透明启发式结果。
- 默认语言：规格书、恢复方案、变更日志、审查结论、交接说明和新增注释默认中文。

## 架构决策
| 日期 | 决策 | 原因 |
|---|---|---|
| 2026-06-29 | 市场概率与上下文调整概率并列保存 | 避免启发式算法覆盖可复核的原始市场定价 |
| 2026-06-29 | 回测强制校验数据截止时间早于开赛 | 防止赛后信息泄漏 |
| 2026-06-29 | FIFA官方匿名接口负责赛程、比分和球员进球，Polymarket公开Gamma API负责90分钟市场价格 | 无需密钥；事实数据与市场数据职责分离 |
| 2026-06-29 | 实时失败不回退旧JSON | 避免把过期数据误标为实时结论 |
| 2026-06-29 | 匿名公开接口未验证到结构化伤停时将覆盖标记为false | 宁可缺失也不推测或把空列表解释成确定无伤 |
| 2026-06-29 | 中国体彩网指定 `hhad,had` 接口负责可购买状态与官方HAD/HHAD即时SP，Polymarket仅保留为欧赔参考 | 体彩与预测市场口径不同，必须统一管理但独立展示和解释 |
| 2026-07-02 | 单场和批量预测必须共用完整融合构建器并输出流程覆盖状态 | 防止批量入口跳过总进球、半全场、凯利、上下文和情报 |
| 2026-07-02 | 训练记录强制满足预测时间早于开赛且结算时间不早于开赛 | 阻断测试mock、提前赛果和时间泄漏污染语料 |
| 2026-07-04 | 命中与失误样本全部参与训练，少于500场只评估不激活校准参数 | 修复只学习失误的选择偏差，防止小样本过拟合 |
| 2026-07-04 | 冠军榜使用公开夺冠赔率强度与已确认16强固定对阵树 | 区分可复核的市场基准、路径概率与单场预测模型 |
| 2026-07-05 | 2026世界杯只能进入封存预序测试，参数仅由2022世界杯与赛前近两年数据选择 | 防止同届赛事既调参又报告测试成绩；允许已结束场次按顺序更新球队状态 |
| 2026-07-05 | 加时和点球赛事使用ESPN分节比分还原90分钟标签 | 保证胜平负、总进球和比分训练口径与体彩90分钟结算一致 |
