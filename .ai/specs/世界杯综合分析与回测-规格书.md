# 世界杯综合分析与回测技术规格书

## 目标
- 在现有赔率去水分析基础上，加入世界杯比赛规则、球队小组赛数据、球员伤停/停赛和主要进球信息，输出透明可解释的综合判断。
- 使用 2026-06-28 南非 vs 加拿大的开赛前快照回测，验证预测链路、防数据泄漏与赛果评价功能。

## 核心口径
- 原始市场概率必须保留，不得被上下文概率覆盖。
- 上下文调整为固定权重启发式：球队积分效率、净胜球效率、进球效率、缺阵负担形成主客强弱信号；以有限幅度对市场概率做乘法调整后重新归一化。
- 主要射手信息用于解释进攻来源，不重复计入数值评分，避免和球队进球数据双重计算。
- 淘汰赛预测仍为 90 分钟胜平负；加时和点球规则只作为规则说明，不推断最终晋级概率。
- 回测只允许读取 `actual_result` 做命中评价；预测函数不得使用该字段。
- 单场回测只称为冒烟验证，不得声称具有统计显著性或证明长期准确率。

## 影响范围
| 文件 | 原因 |
|---|---|
| `worldcup_mvp/analyzer.py` | 增加上下文校验、信号计算、综合概率、规则说明和回测评价 |
| `worldcup_mvp/__init__.py` | 导出新增回测接口 |
| `main.py` | 增加综合分析展示和 `--backtest` 模式 |
| `data/matches_2026-06-29.json` | 补充今日三场赛前球队/人员/进球/规则数据及来源 |
| `data/backtest_2026-06-28.json` | 新增南非 vs 加拿大赛前快照与赛后评价字段 |
| `tests/test_analyzer.py` | 覆盖综合信号、归一化、兼容旧数据、回测命中和时间防泄漏 |
| `README.md` | 更新输入格式、命令、算法边界和回测说明 |

## 实施步骤
1. 兼容读取旧赔率 JSON；上下文字段缺失时退化为原市场概率。
2. 校验球队统计、缺阵影响值、赛事规则和数据时间。
3. 计算四类透明信号并限制调整幅度，输出每项贡献和综合概率。
4. 生成按规则、状态、攻防、人员、进球点、市场校准组织的中文分析过程。
5. 新增回测接口和命令，输出预测、实际结果、是否命中及样本限制。
6. 补充今日三场与昨日回测数据，所有外部事实附来源 URL 和截止时间。
7. 运行单元测试、今日分析和回测命令。

## 验证方式
- `python -m unittest discover -s tests -v`
- `python main.py`
- `python main.py --json`
- `python main.py --file data/backtest_2026-06-28.json --backtest`
- 检查所有概率和为 1、赛果字段不改变预测、截止时间不早于开赛时拒绝回测。

## 回滚方案
- 删除新增回测数据文件，恢复目标文件到改动前版本；旧格式赔率文件必须始终保持可运行。
