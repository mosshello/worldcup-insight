# 近期首发与可用性情报 技术规格书

## 目标
- 从匿名公开比赛摘要中提取两队最近一场确认首发和技术统计，为当日世界杯分析提供可审计的预计首发与阵容稳定性信号。

## 影响范围
| 文件 | 原因 |
|---|---|
| `worldcup_mvp/recent_lineups.py` | 新增 ESPN 匿名公开 JSON 适配、首发解析与最近比赛匹配 |
| `worldcup_mvp/match_intelligence.py` | 将近期首发和可用性信号并入现有情报报告 |
| `tests/test_recent_lineups.py` | 覆盖比赛匹配、首发解析和降级行为 |
| `tests/test_match_intelligence.py` | 覆盖情报报告合并行为 |
| `worldcup_mvp/sporttery_api.py` | 修复赛程接口与在售列表不一致时的固定奖金回填 |
| `tests/test_score_predictor.py` | 覆盖详情已开盘但在售列表漏项的场景 |
| `.ai/project.md` | 记录数据边界和变更结果 |

## 实施步骤
1. 通过固定 HTTPS 域名读取 ESPN 世界杯 scoreboard 与 summary JSON，设置超时、输出上限和失败降级。
2. 对已完场比赛按球队与时间排序，只取最近一场，提取确认首发、位置、阵型位和球队统计。
3. 今日预计首发默认沿用上一场首发，明确标注为“近期阵容推断”，不将未查到伤停解释为官方确认健康。
4. 在情报覆盖中分开记录 `recent_lineup_inference` 与 `injury_api`，保留原有 overlay 的更高优先级。
5. 对赛程状态为 Selling 但计算器列表漏项的场次，以单场固定奖金详情中的 HAD/HHAD 回填可分析状态。

## 验证方式
- `python -m unittest tests.test_recent_lineups tests.test_match_intelligence -v`
- `python -m unittest discover -s tests -v`
- 对今日世界杯场次运行 `python predict.py slate --json`，核对情报中的上一场首发和来源时间。

## 回滚方案
- 删除新适配器并恢复 `match_intelligence.py` 的原有情报构建；overlay 和市场概率不受影响。
