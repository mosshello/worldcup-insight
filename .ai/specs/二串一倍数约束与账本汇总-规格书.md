# 二串一倍数约束与账本汇总技术规格书

## 目标
- 二串一必须来自模型稳定候选，组合 SP 不低于 2.00；页面展示累计总投入、已结算盈利、未结算潜在盈利与历史方案。

## 影响范围
| 文件 | 原因 |
|---|---|
| `worldcup_mvp/daily_bet.py` | 新增满足最低组合赔率的最优二串一选择与账本汇总 |
| `tests/test_daily_bet.py` | 覆盖 2.00 下限、联合概率排序与累计统计 |
| `data/daily_bets.json` | 重算今日二串一并补充结算占位字段 |
| `scripts/generate_pages.py` | 展示累计指标和历史记录 |
| `docs/index.html` | 更新静态页面 |
| `docs/daily_bets.json` | 更新公开账本快照 |
| `README.md` | 说明组合规则与盈亏口径 |
| `.ai/project.md` | 更新变更日志 |

## 实施步骤
1. 从同一销售日全部高信心、内外盘同向预测中枚举两两组合。
2. 过滤组合 SP 小于 2.00 的组合，在剩余组合中选择联合去水概率最高者；并列时选择赔率更低者。
3. 稳健单场仍取去水概率最高方向，不要求必须成为二串一腿。
4. 汇总所有历史条目的总投入；已结算盈利只统计明确写入 `realized_pnl` 的条目；未结算潜在盈利按单场和二串一潜在利润之和展示。
5. 页面增加汇总卡与按日期排列的历史方案，明确“待结算”不计入已实现盈利。

## 验证方式
- `python -m unittest tests.test_daily_bet`
- `python -m unittest discover -s tests`
- `python scripts/generate_pages.py`
- 浏览器检查累计指标、2倍标签和历史记录。

## 回滚方案
- 恢复前两名直接组合逻辑与旧页面生成器，重新生成账本及 Pages 文件。
