# 全流程预测与训练数据治理技术规格书

## 目标
- 统一批量与单场预测入口，确保每次预测都执行体彩主盘、外部参考、走势、总进球/半全场、凯利、FIFA上下文和赛前情报流程。
- 阻止开赛前或测试模拟赛果进入训练语料。
- 删除重复入口与 Python 缓存产物，保留用于验收主流程的测试。
- 提供可执行的 Elo/Poisson 训练方案和数据门槛。

## 实施步骤
1. 在 `dashboard_data.py` 抽取基于已匹配场次的完整融合构建器，单场和批量入口复用。
2. 批量输出保留原卡片字段，同时附带融合预测、盘口分析、上下文、情报和覆盖状态；缺少非核心数据时显式标记。
3. 将记录预测和每日模拟记账放在完整批量流程结束后，避免部分流程提前写入。
4. 删除重复批量脚本，在 `predict.py` 增加 `slate` 与 `audit-training`。
5. 训练记录强制校验预测时间早于开赛、结算时间不早于开赛、赛果字段完整。
6. 修复结算测试的真实文件污染，并删除已经污染的本地运行数据。
7. 清理缓存目录，更新文档。

## 验证方式
- `python -m unittest discover -s tests -v`
- `python predict.py slate --json`
- `python predict.py audit-training --json`
- `python predict.py predict --match-id <可售场次ID> --json`

## 回滚方案
- 恢复 `dashboard_data.py` 原批量调用和 `scripts/today_score_predict.py`。
- 训练语料校验只拒绝新增记录，不迁移或改写已确认的历史记录。
